# Key Decisions

All major decisions made during the research phase (April 2026). Each decision is research-backed — see `docs/decisions/open-questions.md` for the full Q&A log with sources.

- **Product name**: CrawlReady
- **Core mechanic**: dual detection — server-side user-agent detection (today) + `Accept: text/markdown` content negotiation (future-proof) + serving AI-optimized content
- **Onboarding model**: graduated (free diagnostic → CDN snippet → DNS proxy), not DNS-first
- **Positioning (revised — April 2026 strategic review)**: "The AI interface layer — the interface between your website and AI, for crawlers, visual agents, commerce agents, and compliance." Four markets, one architecture: GEO/AI Search ($1.5B), Agentic AI ($7.6B), AI Compliance (EU AI Act Aug 2026), Traffic Defense (immediate).
- **Phase 0 (scoped)**: build exactly 2 things — landing page + working diagnostic — plus the AI Crawler Analytics ingest endpoint. Diagnostic shows a unified AI Readiness Score (0-100) as headline, with Crawlability + Agent Readiness + Agent Interaction sub-scores on drill-down + EU AI Act readiness checklist. Pre-seed 20 sites (not 200). Show HN as sole launch channel.
- **Phase 0 ICP (broadened April 7, 2026)**: All JS-heavy sites — CSR, SSR with heavy client components, and hybrid — whose sites look different to AI crawlers than to humans. Use CSR invisibility as the most dramatic *example* in marketing (Show HN, blog screenshots), but don't limit addressable market to pure CSR SPAs (<5% of new sites in 2026).
- **Growth engine**: public score URLs (built into diagnostic automatically) + Show HN (Phase 0), then AI Crawler Analytics dashboard + middleware snippets + npm package + MCP server + Docusaurus plugin + badges + hidden backlink engine (Phase 1), then monitoring tool partnerships (Phase 1.5), then open-source + GitHub Action (Phase 2)
- **Differentiators (updated April 7, 2026 critical analysis)**: **Genuinely unique:** (1) free visual diff diagnostic (no competitor shows browser render vs. AI crawler HTML side-by-side with invisible content highlighted), (2) public shareable score URLs (no competitor generates permanent, indexable score pages), (3) compound distribution engine (npm + MCP + badge + GitHub Action + Docusaurus plugin — no competitor has multi-channel developer distribution). **Differentiated but not unique:** unified AI Readiness Score (AgentReady.tools uses same name, similar approach), AI Crawler Analytics (MachineContext has edge-level analytics — CrawlReady differentiates via free framework-level middleware), attribute-rich Schema.org generation (MachineContext auto-generates generic Schema — CrawlReady differentiates via FAQPage/Product/HowTo from CSR pages), multi-format serving per AI client type, Agent Readiness + Agent Interaction Scores as integrated diagnostic, hidden backlink growth engine, transparency-first architecture, EU AI Act compliance alignment, MCP server for IDE-native distribution. Open-source pipeline deferred to Phase 2.
- **Architecture**: API-first — Next.js API routes (`/api/v1/*`) serve as the core backend consumed by all distribution channels (web, MCP server, npm CLI + middleware, Docusaurus plugin, partner referral tracking, edge proxy). CLI and middleware are two distinct concerns within the same npm package — see `docs/architecture/api-first.md`.
- **License**: AGPL for transformation pipeline — **deferred to Phase 2**. Phase 0-1 ships proprietary to maximize velocity. Community management overhead incompatible with 15-20 hrs/week solo.
- **Domain**: crawlready.app (secured)
- **Phase 0 crawling**: Firecrawl API (no self-hosted Playwright). Custom Playwright pipeline only for Phase 2.
- **Pricing**: Starter $29/mo (500 fresh crawls), Pro $49/mo (2,500 fresh crawls), Business $199/mo (10K fresh crawls). Two-unit model: unlimited cached responses + metered fresh crawls. Original limits reduced 10x due to Firecrawl COGS analysis.
- **Unit economics**: Firecrawl at $0.01-$0.05/page. Original crawl limits (5K/25K/100K) made every tier negative margin. Revised limits validated for positive margins. Validate actual per-page costs during Phase 0.
- **Revenue targets**: $3-5K MRR at 12 months (base-to-realistic). $5-8K MRR upside if agent readiness + compliance + monitoring partnerships validate. Base case preserved from original $2-3K; upside raised due to expanded TAM and zero-CAC distribution channels.
- **Cloudflare**: shipped "Markdown for Agents" (Feb 2026) — content negotiation via `Accept: text/markdown`. AI crawlers don't send the header yet (re-confirmed April 2026). Apps marketplace deprecated (June 2025); Worker template deferred to Phase 2.
- **Citation monitoring (Phase 3)**: buy/partner (Otterly API or thin wrapper), do not build from scratch.
- **llms.txt**: demoted to minor feature (no AI engine uses it as of April 2026)
